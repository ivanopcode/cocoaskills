# Контрибуция

Перевод английской версии. Источник правды: [CONTRIBUTING.md](CONTRIBUTING.md).

## Настройка окружения

Требуются Python 3.11+ и git.

```bash
git clone https://github.com/ivanopcode/cocoaskills.git
cd cocoaskills
python -m venv .venv
source .venv/bin/activate
python -m pip install -e ".[dev]"
pytest
```

Тестовый набор строит одноразовые git-репозитории и гоняет настоящий конвейер
установки на временных хранилищах; сетевой доступ не требуется.

## Соглашения по коду

- Runtime-пакет использует только стандартную библиотеку. Новые
  runtime-зависимости сначала проходят обсуждение дизайна.
- Следуйте окружающему коду: раскладка модулей, типы ошибок на модуль,
  dataclass-модели и явная валидация с actionable-сообщениями.
- Имена, становящиеся путями файловой системы, проходят через
  `identifiers.py`.
- Всё, что разбирает сторонний ввод (манифесты, архивы, URL), валидирует до
  действия и никогда не исполняет код из манифеста.
- Тесты идут вместе с изменением. Фикстуры команд поставляют обе входные
  точки, `unix_path` и `win_path`; платформенные проверки несут явные
  платформенные маркеры. CI гоняет Linux, macOS и Windows на поддерживаемых
  версиях Python.

## Коммиты и pull request

- Сообщения коммитов: английский, повелительное наклонение, короткий
  заголовок и тело, объясняющее изменения поведения.
- Одно логическое изменение на коммит; тесты и документация едут вместе с
  изменением.
- Для видимых пользователю изменений обновляйте `CHANGELOG.md` в секции
  `[Unreleased]` (формат Keep a Changelog).

## Изменения дизайна

Изменения поведения или форматов манифестов, разрешения, раскладки установки
или модели аудита начинаются с RFC в `docs/` (`v<target>-design.md`,
нумерованные заголовки RFC, заголовок Status). Реализация следует за принятым
RFC. История RFC собрана в
[ARCHITECTURE.md](ARCHITECTURE.md#design-history).

## Документация

- Английские документы являются источником правды; русские переводы живут
  рядом с суффиксом `.ru.md` и шапкой со ссылкой на оригинал.
- Код-блоки в переводе остаются идентичными оригиналу.
- Стиль избегает длинных тире, кавычек-ёлочек и риторических антитез;
  утверждайте, чем вещи являются.

## Релизы

Мейнтейнеры выпускают релиз обновлением `CHANGELOG.md` (перенос `[Unreleased]`
в секцию версии со ссылками сравнения) и пушем аннотированного тега `v*`. CI
собирает дистрибутивы, публикует на PyPI и создаёт GitHub-релиз с
контрольными суммами и аттестациями. Homebrew-формула в
`ivanopcode/homebrew-csk` бампится после релиза.
